## 2. 기술 스택

| 레이어 | 스택 |
|--------|------|
| **백엔드** | Python 3.11+, FastAPI 0.115, Uvicorn, Pydantic v2 |
| **프론트엔드** | React 18, Vite 5, Chart.js 4, @supabase/supabase-js |
| **STT** | OpenAI Whisper medium (CUDA 우선, CPU 폴백) |
| **포즈 분석** | YOLOv8n (person detection) + MediaPipe Pose 33 landmarks |
| **AI 피드백** | Google Gemini 1.5 Flash (`google-generativeai`) |
| **발음 평가** | Whisper `avg_logprob` 재활용 (별도 모델 없음) |
| **언어 베이스라인** | 자체 학습 선형 회귀 (AI Hub 720개 레코드) |
| **DB/Auth** | Supabase PostgreSQL + Supabase Auth (이메일/Google OAuth) |
| **ORM** | supabase-py 2.0 (직접 SQL, RLS 적용) |

## 4. 각 서비스 파일 역할

| 파일 | 역할 |
|------|------|
| `stt_service.py` | Whisper medium으로 오디오 전사. `avg_logprob` 포함 반환 |
| `language_service.py` | STT 결과 → 피처 추출 → 루브릭 채점 → 베이스라인 보정 → 개선 답변 생성 |
| `pose_service.py` | 영상 → YOLOv8 인물 크롭 → MediaPipe 33 landmark → 자세·시선·손동작 점수 |
| `nonverbal_service.py` | 이벤트 카운트(수동 또는 키포인트) → 비언어 4항목 채점 |
| `pronunciation_service.py` | Whisper `avg_logprob` → 선형 정규화 → 1~5등급 / 0~100점 변환 |
| `fusion_service.py` | 언어(60%) + 비언어(25%) + 발음(15%) 가중 합산 최종 점수 |
| `gemini_service.py` | Gemini 1.5 Flash에 분석 결과 전달 → JSON 형식 AI 피드백 |
| `feedback_service.py` | 룰 기반 강점·약점·권고 생성 (Gemini 폴백) |

## 5. 채점 구조 (가중치 포함)

### 최종 점수 합산

```
최종 점수 = 언어점수 × 60% + 비언어점수 × 25% + 발음점수 × 15%
           (발음 없으면 언어 70.6% + 비언어 29.4% 으로 재정규화)
```

### 언어 점수 내부 구조

```
언어 overall = 루브릭 × 70% + AI Hub 베이스라인 × 30%

루브릭 세부 (overall 기준):
  도입-본론-결론 구조     × 25%
  명확성 (문장 길이·반복)  × 20%
  유창성 (간투어·반복)     × 20%
  질문 적합성 (키워드)     × 20%
  간결성 (발화량)          × 15%
```

### 구조 점수 5등급 (AI Hub 기준)

```
도입 30% + 본론 40% + 결론 30%

각 섹션:
  A (1.00): Primary ≥2개, 또는 Primary ≥1 + Secondary ≥2
  B (0.80): Primary ≥1 + Secondary ≥1
  C (0.60): Primary ≥1만
  D (0.40): Secondary ≥2 (Primary 없음)
  F (0.00): 마커 없음
```

### 비언어 점수 내부 구조 (영상 업로드 시)

```
포즈 분석 (pose_service):
  자세 안정성 × 35%  (어깨·골반 기울기 Gaussian decay)
  시선 안정성 × 35%  (코 x좌표 편차)
  손동작 적절성 × 30% (손목 위치·이동 안정성 + 골반 아래 비율)
```

### 발음 점수

```
avg_logprob → 0~100 선형 정규화
  [-0.1 이상] → 100점 (매우 좋음)
  [-1.5 이하] → 0점   (많은 연습 필요)
```

## 6. 남은 작업 / 미완성 부분

| 항목 | 상태 | 비고 |
|------|------|------|
| ResultPage 발음 점수 표시 | **미반영** | `pronunciation_score_100`, `pronunciation_grade`가 API 응답에는 있으나 ResultPage UI에서 출력 안 함 |
| Gemini 피드백 ResultPage 표시 | **미반영** | `gemini_feedback` 필드 반환은 되나 ResultPage에서 렌더링하지 않음 |
| 비언어 점수 최종 합산 연동 | **부분 미완** | `routes_upload.py`는 포즈 점수를 반환하나 `fusion_service`와 아직 연결 안 됨 (`/upload/audio` 응답에 `nonverbal_score`는 있지만 `final_score` 필드는 없음) |
| `enableNonverbal` 영상 분석 | **UI 하드코딩** | `App.jsx`의 `nonverbalEvents`가 실제 영상 분석이 아닌 수동 preset 값으로 전달됨 |
| 히스토리 상세 결과 복원 | **부분 구현** | 기록 페이지에서 결과 조회 시 전체 응답 구조 복원 검증 필요 |
| 프론트 빌드 | **dist 존재** | 현재 `dist/` 빌드 파일이 있으나 최신 코드 반영 여부 미확인 (`npm run build` 필요) |

## 7. API 엔드포인트 명세

> 인증이 필요한 엔드포인트는 `Authorization: Bearer <access_token>` 헤더 필요.  
> 선택적 인증(`?`)은 비로그인도 분석 가능하지만 로그인 시 기록이 저장됨.

### 업로드 (`/upload`)

| 메서드 | 경로 | 인증 | 설명 |
|--------|------|------|------|
| POST | `/upload/audio` | ? | 오디오/영상 업로드 → 전체 분석 파이프라인 |
| POST | `/upload/video` | ? | `/upload/audio`와 동일 (별칭) |

**`POST /upload/audio` — Query Parameters**

| 파라미터 | 타입 | 기본값 | 설명 |
|----------|------|--------|------|
| `context_mode` | `interview` \| `presentation` | `presentation` | 분석 모드 |
| `expected_duration_sec` | int (≥1) | null | 목표 발화 시간(초) |
| `role` | string | null | 지원 직무 예: `backend developer` |
| `keywords` | string | `""` | 쉼표 구분 핵심 키워드 예: `문제해결,협업` |

**Body:** `multipart/form-data` — `file` 필드 (`.mp3 .mp4 .wav .m4a .webm .ogg .flac`)

**Response: `AudioAnalysisResponse`**

```json
{
  "transcript": "전사문",
  "audio_duration_sec": 92.3,
  "language": { /* LanguageAnalysisResponse */ },
  "history_id": "uuid or null",
  "posture_score": 78.5,
  "gaze_score": 82.1,
  "gesture_score": 65.0,
  "nonverbal_score": 75.4,
  "gemini_feedback": {
    "overall_comment": "...",
    "strengths": ["..."],
    "improvements": ["..."],
    "nonverbal_feedback": "...",
    "language_feedback": "..."
  },
  "pronunciation_score": 4,
  "pronunciation_score_100": 78.5,
  "pronunciation_grade": "좋음"
}
```

> 오디오 파일이면 `posture_score`, `gaze_score`, `gesture_score`, `nonverbal_score`, `gemini_feedback`은 `null`.

### 분석 (`/analyze`)

| 메서드 | 경로 | 인증 | 설명 |
|--------|------|------|------|
| POST | `/analyze/language` | 없음 | 텍스트 언어 분석만 |
| POST | `/analyze/nonverbal` | 없음 | 이벤트 카운트 기반 비언어 분석 |
| POST | `/analyze/nonverbal/keypoints` | 없음 | 키포인트 기반 비언어 분석 |
| POST | `/analyze/full` | ? | 언어+비언어 통합 분석 + 최종 점수 합산 |

**`POST /analyze/language` — Body**

```json
{
  "transcript": "발화 전사문",
  "context_mode": "interview",
  "expected_duration_sec": 90,
  "role": "backend developer",
  "keywords": ["문제 해결", "실행력"]
}
```

**Response: `LanguageAnalysisResponse`**

```json
{
  "overall_score": 74.5,
  "metrics": [
    { "name": "structure", "score": 80.0, "note": "구조" },
    { "name": "clarity", "score": 70.0, "note": "명확성" }
  ],
  "detected_issues": ["간투어 과다 사용"],
  "extracted_features": { "filler_rate": 0.08, "wpm": 142 },
  "model_outputs": { "rubric_score": 72.0, "baseline_grade_score_10": 6.8 },
  "readable_feedback": {
    "summary": "...",
    "strengths": ["..."],
    "weaknesses": ["..."],
    "next_actions": ["..."]
  },
  "improved_answer": "AI 개선 답변 예시"
}
```

**`POST /analyze/full` — Body**

```json
{
  "context_mode": "interview",
  "history_id": "기존-uuid (업데이트용, 선택)",
  "pronunciation_score_100": 78.5,
  "language": { /* LanguageAnalyzeRequest */ },
  "nonverbal": {
    "context_mode": "interview",
    "clip_duration_sec": 90.0,
    "hand_movement_events": 12,
    "head_movement_events": 8,
    "posture_shift_events": 3,
    "meaningless_gesture_events": 2
  }
}
```

**Response: `FullAnalysisResponse`** — `language`, `nonverbal` 각 결과 + `final_score`, `weights`, `readable_feedback` 포함.

## 8. 서버 실행 방법

**백엔드**

```powershell
# 가상환경 활성화
.venv\Scripts\Activate.ps1

# 서버 시작 (포트 8000)
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**프론트엔드 개발 서버**

```powershell
cd app\ui
npm run dev          # http://localhost:5173
```

**프론트엔드 빌드 후 백엔드에서 서빙 (통합 실행)**

```powershell
cd app\ui
npm run build        # dist/ 생성

# 이후 백엔드만 실행하면 http://localhost:8000/ui/ 로 UI 접근 가능
```

**환경변수 (.env 필수)**

```
SUPABASE_URL=...
SUPABASE_KEY=sb_publishable_...        # anon key (브라우저용)
SUPABASE_SERVICE_ROLE_KEY=sb_secret_...  # service role (서버 전용, VITE_ 절대 금지)
GEMINI_API_KEY=...
WHISPER_MODEL=medium                   # 기본값
```
